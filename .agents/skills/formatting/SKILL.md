## Formatting Code

1. Use `npm run format` to format the codebase with Prettier.
2. Use `npm run lint` to check for linting errors and fix them.
3. Use `npm run check` to check for TypeScript type errors and fix them.

## Best Practices

- Always run the formatting and linting commands before committing code.
- Ensure that all formatting and linting issues are resolved before opening a pull request.
